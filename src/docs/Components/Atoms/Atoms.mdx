## What is Atomic Design?

**Atomic Design** is a methodology created by Brad Frost that organizes interface components into **5 hierarchical levels**, working together to create design systems in a deliberate and structured way.

> "Atomic Design is not a linear process, it's a mental model for thinking about interfaces as both a cohesive whole and a collection of parts at the same time."

## The 5 Levels of Atomic Design

### 1. ğŸ§ª **Atoms**
**Basic elements that cannot be further decomposed**

Atoms are the fundamental building blocks of the interface. In Draupnir, we have:

- **Avatar** - Visual representation of users
- **Badge** - Status and category indicators  
- **Box** - Flexible container for layouts
- **Button** - Interactive actions with variants
- **Divider** - Visual separators
- **Text/Typography** - Typography system

```jsx
// Example of atom usage
<Button variant="primary">Save</Button>
<Badge variant="success">Active</Badge>
<Typography size="lg">Title</Typography>
```

### 2. ğŸ§¬ **Molecules**
**Collections of atoms that form simple components**

Molecules combine atoms to create specific functionalities:

- **Input** - Input field with prefix/suffix
- **Link** - Navigation with states
- **Loading** - Loading indicators
- **Notice** - Notifications and alerts
- **Select** - Selection with dropdown
- **Switch** - State toggles

```jsx
// Example of molecule (Input compound)
<Input.Root>
  <Input.Prefix>
    <Icon /> {/* Atom */}
  </Input.Prefix>
  <Input.Control placeholder="Type here..." />
  <Badge variant="error">Error</Badge> {/* Atom */}
</Input.Root>
```

### 3. ğŸ¦  **Organisms**
**Complex components that form distinct sections**

Organisms combine molecules and atoms to create functional sections:

- **Dropdown** - Dropdown menu with options
- **Header** - Application header
- **Modal** - Modal windows with content
- **SubtitlePage** - Pages with subtitles
- **TitlePage** - Pages with titles

```jsx
// Example of organism (Header)
<Header>
  <Avatar size="sm" image="/logo.png" /> {/* Atom */}
  <nav>
    <Link href="/dashboard">Dashboard</Link> {/* Molecule */}
    <Link href="/profile">Profile</Link>
  </nav>
  <Button variant="secondary">Logout</Button> {/* Atom */}
</Header>
```

### 4. ğŸ“‹ **Templates**
**Layouts that demonstrate the design structure**

Templates define the structure and organization of components:

- **Page layouts** - Base structures for different page types
- **Responsive grids** - Content organization
- **Sidebars and navigation** - Navigation structures

### 5. ğŸ“„ **Pages**
**Concrete implementations with real content**

Pages apply real content to templates and test design resilience:

- **Login/register pages**
- **Dashboards**
- **Complex forms**
- **Data listings**

### ğŸ”„ **Construction Flow**

Draupnir follows the **mental model** of Atomic Design:

1. **Think from whole to parts** - We start by analyzing complete pages
2. **Break into smaller units** - We break down into organisms, molecules, and atoms
3. **Build from bottom up** - We develop atoms first, then molecules, etc.

### ğŸ¯ **Applied Principles**

#### **Component Atomicity**
Each component is **complete on its own**:
- Independent of business rules
- Not coupled to dynamic data
- Works in isolation
- Can be tested independently

#### **Flexible Composition**
- **Compound Components** - Like `Input.Root`, `Input.Control`, `Input.Prefix`
- **Props composition** - Flexible variants and configurations
- **Smart reuse** - One atom can be used in multiple molecules

#### **Access Hierarchy**
```

```

Each level can access **any level below**, but never above.

### ğŸ¨ **Intentional Design System**

Draupnir is an **intentional Design System** that:

- **Defines visual patterns** - Colors, typography, spacing
- **Centralizes tokens** - Reusable values in `src/tokens/`
- **Documents components** - Storybook for each level
- **Facilitates maintenance** - Changes in atoms reflect throughout the application
